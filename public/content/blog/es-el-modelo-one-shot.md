# El Modelo One-Shot

## Por qué la Infraestructura Legal No Es SaaS

---

## Tesis

La mayoría de las soluciones legales digitales están construidas bajo una premisa incorrecta: que el valor reside en el acceso continuo al sistema. El modelo one-shot parte de una premisa distinta: el valor no está en el acceso, el valor está en la ejecución irreversible. Esta diferencia no es semántica, es arquitectónica, económica y doctrinal.

---

## 1. El Error del Modelo SaaS Aplicado al Derecho

El modelo SaaS fue diseñado para retener usuarios, construir cuentas, acumular datos, maximizar el tiempo dentro del sistema y monetizar la continuidad. Ese modelo funciona perfectamente para sistemas como CRM, gestión de proyectos, herramientas colaborativas y plataformas de contenido.

Pero el acto jurídico no es continuo, es puntual. Un abogado no "usa" una demanda, la presenta. Un profesional no "mantiene" una contestación, la ejecuta. El derecho procesal no es una suscripción, es una secuencia de actos formales con consecuencias. Aplicar SaaS al litigio es forzar una estructura económica ajena a la naturaleza del acto jurídico.

---

## 2. El Acto Jurídico Como Evento Irreversible

En el modelo one-shot, el sistema no ofrece acceso permanente. Ofrece una ejecución, un borrador, una descarga y una destrucción total posterior. No hay historial, biblioteca, proyectos, versionado persistente ni reanudación futura.

El sistema no acumula memoria, no construye relación y no conserva evidencia. Opera bajo un principio simple: una ejecución produce un artefacto, el artefacto se descarga y todo lo demás desaparece.

---

## 3. De Plataforma a Máquina

Un SaaS es una plataforma. Una plataforma tiene usuarios, cuentas, continuidad y retención como métrica central. El modelo one-shot no es plataforma, es máquina. Una máquina se activa, ejecuta, produce y se apaga.

No existe "uso continuo", existe activación puntual. El sistema no busca que el profesional permanezca dentro, optimiza para que salga lo antes posible con un resultado válido. La métrica no es tiempo en el sitio, es tiempo hasta descarga.

---

## 4. Pago Como Autoridad, No Como Acceso

En el modelo SaaS se paga por acceso, por membresía y por continuidad. En el modelo one-shot se paga por autoridad de ejecución.

El pago no habilita una cuenta, habilita un único cruce irreversible. Una vez consumida esa autoridad, no puede reutilizarse, no puede extenderse y no puede reiniciarse. No existen créditos, no existe saldo y no existe wallet. Cada ejecución es un evento económico aislado.

---

## 5. Eliminación Como Modelo de Confianza

El SaaS tradicional construye confianza prometiendo seguridad de almacenamiento, historial recuperable, backup y control de acceso. El modelo one-shot construye confianza por ausencia: no hay almacenamiento, no hay historial, no hay datos persistentes, no hay recuperación posterior.

La información no se guarda "segura", se destruye. Esto no es un "feature", es el núcleo del modelo económico. Si no hay datos retenidos, no hay superficie de explotación futura.

---

## 6. Consecuencia Estratégica

El modelo one-shot implica que no existe retención como KPI, no existe LTV acumulativo por usuario, no existe lock-in y no existe ecosistema cerrado.

La sostenibilidad no proviene de la dependencia del cliente, proviene de la claridad del acto. Cada ejecución es independiente, cada sesión termina en incineración y cada interacción es autosuficiente.

---

## 7. Lo Que Este Modelo No Es

No es una suite legal, un workspace, un gestor documental, un asistente continuo, un entorno colaborativo ni un repositorio de conocimiento.

Cualquier movimiento hacia esas categorías crea otro negocio distinto.

El modelo one-shot es deliberadamente restrictivo. La restricción es su fortaleza.

---

## 8. Arquitectura Como Doctrina

Este modelo no es una preferencia de UX.
Es una doctrina operacional.

Define:

- Cómo se cobra.
- Cómo se ejecuta.
- Qué se permite.
- Qué se prohíbe.
- Qué se destruye.

La arquitectura impone límites que el marketing no puede suavizar.

No se vende relación, no se vende permanencia, no se vende "uso".

Se vende un acto.

---

## Conclusión

El derecho procesal es irreversible por naturaleza y el modelo one-shot simplemente respeta esa estructura. Donde el SaaS busca permanencia, el modelo one-shot busca finalidad. Donde el SaaS acumula, el modelo one-shot elimina. Donde el SaaS retiene, el modelo one-shot ejecuta y desaparece. No es software que se usa, es una ejecución que se autoriza. Y termina.
